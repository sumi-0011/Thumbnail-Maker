## 제목 타입별 생성 규칙

### narrative (서술형)
```
구조: [주어/주제] → [동사/관점] → [목적/대상] → [보충] → [이모지?]
```
- 태그를 이어 읽으면 완전한 문장
- 분석의 `writing_motivation`을 문장 흐름으로 변환
- hook_type이 있으면 도입부 뉘앙스 반영
- 조사는 앞 단어에 붙임: "관점으로", "바라보는"
- 태그 4~6개, 각 2~12자

### question (질문 후킹형)
```
구조: [질문 태그] → [이모지] → [답/주제 태그들]
```
- 분석의 `hook`을 질문으로 변환 (hook_type이 problem/curiosity면 최적)
- 질문은 독자의 실제 고민을 반영: "이럴 때 뭐 쓰지?", "왜 느릴까?"
- 질문에 반드시 `?` 포함
- 질문 뒤에 🤔, 💭, ❓ 등 이모지
- 답은 `key_terms` + `core_topic`에서 추출
- 태그 4~5개

### keyword_stack (키워드 스택형)
```
구조: [핵심 키워드] → [키워드] → [키워드] → [관계어] → [키워드] ...
```
- `key_terms`를 그대로 활용 (영어 원문 유지)
- 관계어 1~2개로 맥락 부여: "장점", "차이점", "비교", "정리", "활용"
- 가장 대표 키워드를 1번째에 배치
- 태그 5~7개, 각 1~8자

### subtitle (부제 포함형)
```
구조: [메인 태그들] → [": 부제"]
```
- 메인 태그: `core_topic` + `key_terms`로 구성 (3~5개)
- 부제 생성 규칙 (아래 3가지 중 택 1):
  - **방법형**: `takeaway`에서 구체적 기법 추출 → ": keepalive 옵션 활용"
  - **교훈형**: `takeaway`를 압축 → ": 복잡도를 줄이기 위한 전략"
  - **감성형**: `emotion`이 reflective/humorous일 때 → ": 그래도 재밌었잖아 한잔해"
- 부제는 반드시 `: `로 시작하는 별도 태그
- 부제 8~20자

### retrospective (회고/감성형)
```
구조: [시간] → [활동/주제] → [키워드들] → [감성 마무리]
```
- 분석의 `intro.hook`에서 시간 표현 추출 → 첫 태그
- `key_terms`에서 고유명사(행사명, 프로젝트명) 추출
- `conclusion.emotion` + `takeaway`로 감성 마무리 태그 생성
- 부제형과 결합 가능
- 구어체/반말 허용
- 태그 4~6개

### provocative (도발/놀라움형)
```
구조: [놀라운 조합/주장] → [이모지?] → [구체적 내용]
```
- 분석의 `unique_angle`에서 의외성 추출
- 첫 태그에 `?!`, `~도?`, `실화?` 등 놀라움 표현
- 이모지로 텍스트에 직접 붙이기 가능: "Spring🌱"
- 과장 금지 — 실제 내용 기반
- 태그 3~5개

### highlight (핵심 강조형)
```
구조: [핵심 키워드] → [이모지] → [세부 주제] → [이모지] → [결론]
```
- `key_terms`의 1번째가 핵심 키워드 (가장 큰 태그)
- 이모지를 키워드 사이사이에 배치하여 시각적 리듬
- 마지막 태그는 행동/관점: "비용 절감", "하드웨어 관점", "실전 가이드"
- 태그 3~5개

### casual (캐주얼/재미형)
```
구조: [재미 요소] → [이모지] → [주제] → [친근한 설명]
```
- 의성어/의태어 활용: "뚝딱뚝딱", "슥슥", "차근차근"
- `core_topic`을 친근하게 풀어쓰기
- 이모지 2~3개 적극 사용
- 구어체: "~해보자", "~만들어보는", "~인데요"
- 태그 3~5개

## 생성 프로세스

1. `recommended_types`에서 추천된 2~3개 타입 확인
2. 각 타입별로 1개씩 제목 생성 (총 2~3개 제목)
3. 사용자 선택 스타일이 있으면 해당 타입으로 추가 1개 생성
4. 각 제목의 태그는 `intro`, `body`, `conclusion` 분석 결과를 조합하여 구성

## 활용 우선순위

| 분석 필드 | 우선 활용 대상 |
|-----------|---------------|
| `hook` | question, provocative의 첫 태그 |
| `writing_motivation` | narrative의 문장 흐름 |
| `core_topic` | 모든 타입의 중심 키워드 |
| `key_terms` | keyword_stack, highlight의 태그 구성 |
| `unique_angle` | provocative의 놀라움 요소, subtitle의 부제 |
| `takeaway` | subtitle의 부제, retrospective의 마무리 |
| `emotion` | retrospective/casual의 톤 결정 |

## 공통 규칙

### 한영 혼용
- 기술 용어(React, Docker, API, Spring)는 영어 유지
- 설명/동사/조사는 블로그와 동일 언어
- 하나의 태그 내 혼용 가능: "API 요청하기", "Spring🌱"

### 이모지
- 텍스트에 직접 붙이기: "Spring🌱", "CS ⚙️", "Slack API 연동💬"
- 독립 태그: "🤔", "✨", "🚀"
- 문맥 전환점에 배치 (질문→답, 주제→설명)
- 최소 0개, 최대 3개

### 태그 길이
- 일반 텍스트: 2~12자
- 영어/혼용: 2~16자
- 부제(`:` 포함): 8~20자

### 태그 개수
- 총 3~7개 (이모지 포함)
- 이상적: 4~5개
