## Step 1: 도입부 분석 (Introduction Scan)

글의 첫 20%를 읽고 다음을 추출:

- **hook**: 독자를 끌어들이는 요소. 아래 중 하나를 식별:
  - 문제 제기: "~할 때 어떻게 해야 할까?", "~가 안 돼서 고생했다"
  - 호기심 유발: "~라는 걸 알고 계셨나요?", "의외로 ~"
  - 공감 형성: "다들 한번쯤 ~해본 적 있을 것이다"
  - 선언/도전: "~를 해보기로 했다", "~에 도전했다"
  - 없음: 바로 본론 진입 (이 경우 본문에서 hook 추출)

- **writing_motivation**: 저자가 이 글을 쓴 동기 1문장
  - "브라우저 종료 시 API 요청이 유실되는 문제를 해결하고 싶어서"
  - "Java와 Kotlin 중 어떤 걸 쓸지 고민하는 사람들을 위해"

## Step 2: 본문 분석 (Body Scan)

글의 핵심 내용을 분석하고 추출:

- **core_topic**: 글의 중심 주제 1문장 (20자 이내)
- **key_terms**: 핵심 기술 용어/키워드 3~5개 (영어 원문 유지)
- **unique_angle**: 이 글만의 차별점/고유한 관점 1문장
  - "공식 문서에 없는 실전 트러블슈팅 경험"
  - "하드웨어 관점에서 Context Switching 비용을 분석"
- **content_type**: 글의 종류 판별
  - tutorial: 단계별 가이드, 따라하기
  - comparison: 기술/도구/방법 비교
  - deep-dive: 특정 개념 깊이 파고들기
  - troubleshooting: 문제 해결 과정
  - retrospective: 회고, 경험 정리
  - opinion: 의견, 생각 공유
  - project: 프로젝트 소개, 사이드 프로젝트
  - news: 새 버전, 업데이트 소식

## Step 3: 마무리 분석 (Conclusion Scan)

글의 마지막 20%를 읽고 추출:

- **takeaway**: 핵심 교훈/결론/메시지 1문장
  - "keepalive 옵션으로 브라우저 종료 시에도 안정적인 API 요청이 가능하다"
  - "결국 수요자 중심으로 전환하는 것이 답이었다"
- **emotion**: 마무리의 감정/톤
  - satisfied: 성취감, 만족
  - reflective: 돌아봄, 성찰
  - hopeful: 앞으로의 기대
  - honest: 솔직한 한계 인정
  - humorous: 유머러스한 마무리
  - neutral: 담백한 정리

## Step 4: 제목 타입 추천

위 분석 결과를 종합하여 이 글에 가장 적합한 제목 타입 2~3개를 추천.

**매칭 기준**:
- hook이 질문형 → `question`, `provocative`
- hook이 도전/선언 → `provocative`, `narrative`
- content_type이 comparison → `keyword_stack`, `question`
- content_type이 tutorial → `narrative`, `highlight`
- content_type이 retrospective → `retrospective`, `subtitle`
- content_type이 deep-dive → `highlight`, `subtitle`
- content_type이 project → `casual`, `provocative`
- emotion이 humorous/honest → `retrospective`, `casual`
- key_terms가 4개 이상 → `keyword_stack`
- takeaway가 구체적 기법 → `subtitle`
