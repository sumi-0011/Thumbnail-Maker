# Thumbnail Title System

블로그 글을 분석하여 썸네일 제목을 추천하는 시스템이다.
2단계로 구성된다: **분석** → **제목 추천**

---

## 제목 타입 정의 (8가지)

### 1. 서술형 (Narrative)

**컨셉**: 태그를 순서대로 읽으면 하나의 자연스러운 문장이 되는 구조.

**핵심 규칙**:
- 태그 읽는 순서 = 문장 읽는 순서 (절대로 역순 배치 금지)
- 한국어 조사(은/는/이/가/을/를/에서/으로)는 반드시 앞 단어에 붙임
- 4~6개 태그, 각 태그 2~12자
- 문장의 주어 → 서술 → 목적/결과 순서 유지

**적합한 글**: 튜토리얼, 가이드, 경험 공유, 설명글

**좋은 예시**:
- "프론트엔드" → "개발자 관점으로" → "바라보는" → "관심사의 분리와" → "좋은 폴더 구조" → "✨"
- "Java" → "동시성 프로그래밍" → "🧑‍💻" → "Thread" → "Future" → "Callable"

**나쁜 예시**:
- "관심사" → "프론트엔드" → "바라보는" (순서 뒤섞임)
- "좋은 폴더 구조와 관심사의 분리를 프론트엔드 관점으로 바라보는 법" (하나의 긴 태그)

---

### 2. 질문 후킹형 (Question Hook)

**컨셉**: 독자가 공감할 질문으로 시작해서 답(주제)으로 이어지는 구조.

**핵심 규칙**:
- 첫 1~2개 태그가 질문 (반드시 `?` 포함)
- 질문 직후에 이모지 배치하여 시각적 구분
- 나머지 태그가 답/해결책 제시
- 질문은 독자의 실제 고민을 반영해야 함 (추상적 질문 금지)
- 4~5개 태그

**적합한 글**: 비교글, 추천글, 트러블슈팅, 의사결정 과정

**좋은 예시**:
- "이럴 때 뭐 쓰지?" → "🤔" → "고민을 줄여주는" → "golang" → "패키지 추천" → "📦"
- "브라우저 종료 시" → "☁️" → "안정적으로" → "API 요청하기"

**나쁜 예시**:
- "뭐가 좋을까?" (너무 막연한 질문)
- "React" → "Performance" → "Optimization" (질문 없이 키워드만)

---

### 3. 키워드 스택형 (Keyword Stack)

**컨셉**: 핵심 기술 키워드를 시각적으로 나열하여 한눈에 주제 파악.

**핵심 규칙**:
- 기술 용어는 영어 원문 그대로 유지 (React, Docker, K8s 등)
- 관계를 나타내는 한국어 단어 1~2개 포함 (장점, 차이점, 비교, 정리 등)
- 키워드 간 논리적 그룹핑 (프레임워크끼리, 개념끼리)
- 5~7개 태그, 각 태그 1~8자
- 가장 중요한 키워드를 첫 번째에 배치

**적합한 글**: 기술 비교, 스펙 정리, 치트시트, 개념 나열

**좋은 예시**:
- "Java" → "Kotlin" → "장점" → "차이점" → "Co-Routine" → "확장 함수" → "키워드"
- "DynamoDB" → "AWS" → "localStack" → "걸들인 로컬 테스트"

**나쁜 예시**:
- "Java와 Kotlin의 장단점 비교" (하나의 긴 문장 — 서술형이지 키워드 스택이 아님)
- "A" → "B" → "C" → "D" → "E" → "F" → "G" → "H" (8개 초과, 산만함)

---

### 4. 부제 포함형 (Title + Subtitle)

**컨셉**: 메인 제목 태그들 + 콜론(`:`)으로 시작하는 부제 태그 1개.

**핵심 규칙**:
- 부제 태그는 반드시 `: `(콜론+공백)으로 시작
- 부제는 마지막 태그에 배치
- 부제 역할 3가지 중 하나:
  - (a) 구체적 방법/기술 명시: ": keepalive 옵션 활용"
  - (b) 핵심 교훈/메시지 요약: ": 복잡도를 줄이기 위한 전략 패턴"
  - (c) 감성적 한 줄 코멘트: ": 그래도 재밌었잖아 한잔해"
- 부제 길이: 8~20자
- 메인 태그 3~5개 + 부제 1개 = 총 4~6개

**적합한 글**: 실전 적용기, 회고, 경험담, 특정 기술 딥다이브

**좋은 예시**:
- "브라우저 종료 시" → "☁️" → "안정적으로" → "API 요청하기" → ": keepalive 옵션 활용"
- "디자인 패턴" → "OOP 집합체" → "전략 패턴" → "장단점 분석" → ": 복잡도를 줄이기 위한 전략 패턴"
- "Spring으로도" → "Slack Bot을?!" → "Spring🌱" → "Slack API 연동💬" → ": 사이프론과 함께"

**나쁜 예시**:
- ": 좋은 글" (부제가 너무 짧고 의미 없음)
- "메인 제목 : 부제" (하나의 태그 안에 콜론 — 별도 태그로 분리해야 함)

---

### 5. 회고/감성형 (Retrospective)

**컨셉**: 개인적 경험, 감정, 시간 흐름을 담은 따뜻하거나 솔직한 톤.

**핵심 규칙**:
- 시간 표현으로 시작: 연도, 분기, "N년차", "하반기" 등
- 구체적 경험/활동 키워드 포함 (컨퍼런스명, 프로젝트명, 이벤트명)
- 감성적 마무리 또는 솔직한 감상 포함
- 부제형과 결합 가능 (`: 감성 한 줄`)
- 말투가 딱딱하지 않아야 함 — 일상 대화체 OK
- 4~6개 태그

**적합한 글**: 회고록, 연말 정리, 커리어 이야기, 일상 기록

**좋은 예시**:
- "2024년" → "하반기를 돌아보며" → "디프만" → "SIPE" → "여행" → ": 그래도 재밌었잖아 한잔해"
- "2025 회고" → "왜 작년보다 바빠" → "넥스터즈" → "구름톤" → "정리해고" → "그런데 내년에는 더 바쁠 예정"

**나쁜 예시**:
- "2024년 하반기 회고" → "활동 정리" (건조한 보고서 톤)
- "올해를 돌아보며 다양한 활동을 했습니다" (하나의 긴 태그)

---

### 6. 도발/놀라움형 (Provocative)

**컨셉**: 의외성, 놀라움, 반전을 전면에 내세워 클릭 유도.

**핵심 규칙**:
- `?!`, `~도?`, `~이라고?`, `진짜?`, `실화?` 등 놀라움 표현 사용
- 예상을 깨는 조합 강조 (기존 도구의 새로운 용도, 의외의 결과 등)
- 첫 태그에서 반전 요소 제시
- 과장이 아닌 실제 내용 기반이어야 함
- 3~5개 태그

**적합한 글**: 새로운 시도, 의외의 조합, 실험적 프로젝트, 반직관적 결과

**좋은 예시**:
- "Spring으로도" → "Slack Bot을?!" → "Spring🌱" → "Slack API 연동💬"
- "CSS만으로" → "3D 애니메이션을?!" → "transform 완전정복"

**나쁜 예시**:
- "놀라운 React 성능!!" (근거 없는 감탄)
- "대박 꿀팁 모음!!!" (과장된 어그로)

---

### 7. 핵심 강조형 (Highlight)

**컨셉**: 가장 중요한 키워드/개념을 첫 번째에 크게 배치, 나머지가 보충.

**핵심 규칙**:
- 1번째 태그 = 핵심 주제 (가장 큰 글씨, filled 스타일 권장)
- 2~3번째 태그 = 세부 주제/범위
- 마지막 태그 = 행동/결론 (가이드, 정리, 비교, 실전)
- 이모지는 핵심 키워드 사이에 배치하여 시각적 구분
- 3~5개 태그

**적합한 글**: 특정 기술 딥다이브, 완벽 가이드, 핵심 정리

**좋은 예시**:
- "CS" → "⚙️" → "Context Switching" → "🔄" → "비용 절감" → "하드웨어 관점"
- "React Hooks" → "완벽 가이드" → "실전 예제 포함" → "🚀"

**나쁜 예시**:
- "가이드" → "React" → "Hooks" (핵심 키워드가 뒤에 있음)
- "React Hooks 완벽 가이드 실전 예제 포함" (분리 안 됨)

---

### 8. 캐주얼/재미형 (Casual Fun)

**컨셉**: 친근하고 가벼운 톤으로 접근성을 높이는 구조.

**핵심 규칙**:
- 의성어/의태어 활용: 뚝딱뚝딱, 슥슥, 쓱싹 등
- 구어체/반말 OK: ~해보자, ~만들어보는, ~인데요
- 이모지 적극 활용 (2~3개)
- 기술 용어도 친근하게 풀어쓰기
- 3~5개 태그

**적합한 글**: 사이드 프로젝트 소개, 입문 튜토리얼, 가벼운 팁, 일상 개발

**좋은 예시**:
- "사이드 프로젝트" → "🐱" → "뚝딱뚝딱" → "썸네일을" → "열심히" → "만들어요!"
- "만들어보는" → "나만의 앱" → "😎" → "처음이라 떨리는 중"

**나쁜 예시**:
- "사이드 프로젝트 개발 과정 소개" (너무 딱딱함)
- "ㅋㅋㅋ 개웃긴 버그" (과도한 인터넷 용어)

## 공통 규칙

### 한영 혼용
- 기술 용어(React, Docker, API 등)는 영어 원문 유지
- 설명/동사/조사는 한국어로 작성
- 하나의 태그 안에서 한영 혼용 가능: "Spring🌱", "API 요청하기"

### 이모지 배치
- 텍스트 뒤에 붙이기 가능: "Spring🌱", "CS ⚙️"
- 독립 이모지 태그로 분리 가능: "🤔", "✨"
- 문맥 전환 지점에 배치하면 효과적 (질문→답, 주제→설명 사이)
- 최대 2~3개, 최소 0개

### 태그 길이
- 한국어 태그: 2~12자
- 영어/혼용 태그: 2~16자
- 부제 태그: 8~20자 (`: ` 포함)

### 태그 개수와 레이아웃
- 총 3~7개 태그
- 이상적 구성: 2줄 (첫 줄 2~3개, 둘째 줄 2~3개 + 부제/이모지)
- 첫 줄에 핵심 메시지, 둘째 줄에 보충/감성 요소
